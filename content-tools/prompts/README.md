# content-tools/prompts/

Prompt templates for the offline content pipeline. The runtime app never sees these — they drive the Python scripts in `content-tools/` that produce the reviewed YAML in `content/`.

> Source authorities locked in `planning/05-decision-log.md`:
> - DEC-002 — every bundled question reviewed against the cited verse before inclusion
> - DEC-010 — AI generation is offline only; runtime ships zero LLM keys
> - DEC-011 — KJV is the canonical spelling / verse-validation reference
> - DEC-014 — avatar pool covers all named individuals + peoples/tribes + symbolic animals/objects
> - DEC-015 — Easton's Bible Dictionary is the public-domain seed for the entity list
> - DEC-016 — Jesus excluded; adversarial / genealogy entries included but tagged
> - DEC-018 — alternate spellings are AI-generated then human-reviewed

## Files

| File | Purpose | Feeds task |
|------|---------|------------|
| `question-generation.md` | Generate Bible quiz questions for a pack theme or scope | 1.8 |
| `avatar-blurb.md` | Generate ≤80-char "who they were" blurbs for AvatarEntity records | 1.2 / 1.3 |
| `avatar-aliases.md` | Generate alternate-spelling aliases for AvatarEntity records | 1.4 |
| `non-person-entities.md` | AI-augment the entity list with peoples, animals, symbolic objects | 1.3 |
| `review-checklist.md` | The reviewer-facing prompt shown by the review CLI | 1.9 |

## How to use

Each file is a self-contained template with placeholders in `{{double_braces}}`. The Python generator script reads the file, substitutes placeholders, sends it to the Anthropic API, parses the structured response, writes it to a staging file, and queues it for human review.

## Model & guardrails

- **Model:** Claude Sonnet 4.6 (`claude-sonnet-4-6`) by default — cheap enough for the v1 cost cap, smart enough for scriptural reasoning. Hard upgrade path to Opus 4.7 (`claude-opus-4-7`) for difficult packs.
- **Output format:** YAML or JSON, schema-validated by the generator before staging. Reject + retry on schema violation, up to 3 attempts.
- **Cost cap:** generator script tracks tokens; exits if a per-pack budget (configurable) is exceeded — surfaces this in the run log so the operator can decide whether to top up.
- **Prompt caching:** enable Anthropic prompt caching on the system prompt and the shared canonical reference blocks (Bible book list, KJV scripture excerpts). Per-pack runs reuse the same cached system prompt across hundreds of questions.

## Known failure modes (track these)

- **Hallucinated verse refs.** Mitigation: the review CLI cross-references against the bundled KJV plaintext (task 1.7) — reviewer sees the AI-cited verse rendered from the actual file, not regenerated by the model.
- **KJV-specific names slipping into WEB/ASV-friendly phrasing.** Generation always uses KJV; the reveal-screen translation is handled by the runtime, not generation.
- **Doctrinal inflection.** Quiz questions stay factual ("Who led the Israelites out of Egypt?"), not interpretive ("Why did Moses..."). Enforced in the system prompt and the review checklist.
- **Trick questions / clever distractors that aren't biblical.** Distractor options must be plausible-but-wrong from within scripture itself, not made-up.
